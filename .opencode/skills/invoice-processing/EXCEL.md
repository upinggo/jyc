# Step 5–8: Database Insert, Reply, Summary & Export

This file covers writing validated invoice data to database via `invoice_add` (Step 5), replying to
the user (Step 6), monthly summary generation (Step 7), and export (Step 8).

---

## Step 5: Insert to Database via invoice_add

**Only reached if Step 4 validation passes (all 2 mandatory fields present).**

**IMPORTANT: Check for duplicates before inserting.**

### Database Operations

Use the `invoice_add` MCP tool to insert the invoice data:

| 提取字段 | invoice_add 参数 |
|---------|-----------------|
| 发票号码 | `--number` |
| 开票日期 | `--date` |
| 发票类型 | `--type` |
| 服务项目名称 | `--item` |
| 金额(不含税) | `--amount` |
| 税率 | `--tax-rate` |
| 税额 | `--tax` |
| 价税合计 | `--total` |
| 销售方名称 | `--seller` |
| 销售方税号 | `--seller-tax` |
| 购买方名称 | `--buyer` |
| 购买方税号 | `--buyer-tax` |
| 备注(校验码等) | `--remark` |
| PDF文件路径 | `--attach` |

**Duplicate check:** Before adding, use `invoice_list --number <发票号码>` to check if the invoice already exists. If the result is non-empty, it's a duplicate — skip it.

**Example:**
```
invoice_add --number "INV-2026-0042" --date "2026-04-10" --type "增值税普通发票" \
  --buyer "XX有限公司" --buyer-tax "91110108MA01XXXXXX" \
  --seller "YY有限公司" --seller-tax "91110108MA01YYYYYY" \
  --item "信息技术服务" --amount 1000.00 --tax-rate "6%" --tax 60.00 \
  --total 1060.00 --remark "校验码:12345678901234567890" \
  --attach "invoice_2026-04/INV-2026-0042.pdf"
```

The `invoice_add` tool internally handles SQLite write and file attachment storage.

## Step 7: Monthly Summary (when requested)

When the user asks to summarize a month's invoices:

The monthly summary is generated by invoice MCP tools (`invoice_export` or `invoice_close`).
The output Excel uses the IIT deduction claim format with predefined categories.

**IMPORTANT:** For customization, the user can modify the generated `invoice_summary_*.xlsx`
after export, but the next `invoice_close`/`invoice_export` will overwrite it.

**Output structure (Sheet1 of generated Excel):**
- Row 1: Title "Bills for Expat IIT deduction"
- Row 4: "Handed in by:" + name
- Row 6: "Month:" + month (e.g., "April of 2026")
- Row 9: Headers (DATE, AMOUNT)
- Rows 10-37: Categories with DATE and AMOUNT columns:
  - Row 10: Laundry
  - Row 12: Food & Meals
  - Row 14: Rental fee (**FIXED VALUE — do NOT modify**)
  - Row 16: Airticket
  - (other rows available for additional categories)
- Row 38: Total (SUM formula, auto-calculates)
- Row 43: Date & Signature

**IMPORTANT:** Row 14 (Rental fee) contains a fixed value set by the user.
The invoice MCP tools (`invoice_close`/`invoice_export`) handle this automatically.

### Process

1. Determine the target month (from user message or default to current month)
2. Verify the invoice database exists (`.invoice/` directory)
3. Query invoices using `invoice_list --month YYYY-MM`
4. Categorize each invoice by its 服务项目名称 (service/item name):
   - 餐饮/餐费/食品/外卖 → Food & Meals (row 12)
   - 洗衣/干洗 → Laundry (row 10)
   - 房租/租金/租赁 → Rental fee (row 14)
   - 机票/航空 → Airticket (row 16)
   - Other categories → use available rows (17-37)
6. Generate the summary using a monthly close or export:
   - **With closing:** `invoice_close --month YYYY-MM` (generates Excel + ZIP archive)
   - **Without closing (query only):** `invoice_list --month YYYY-MM` (list invoices)
   - **Export report:** `invoice_export --month YYYY-MM --output ./`
   ```
   This creates `invoice_summary_<month>.xlsx` (via invoice_export) in the current directory.

7. Reply with a summary of categorized amounts

IMPORTANT: The output Excel is generated by invoice MCP — do NOT modify the generated file structure manually.

## Step 8: Export Monthly Invoices (when requested)

When the user asks to download or export all invoices for a month:

1. Determine the target month (from user message or default to current month)
   - Example: `MONTH="2026-04"` or `MONTH="2026-05"`
2. Verify the folder exists
3. Generate the invoice list Excel using invoice_export:
   ```
   invoice_export --month YYYY-MM --output ./
   ```
   This creates `invoice_list_<month>.xlsx` in the current directory.
4. For monthly summary (if needed):
   - **With closing:** `invoice_close --month YYYY-MM` (generates both Excel file + ZIP archive)
   - **Without closing:** Use `invoice_export --month YYYY-MM --output ./`
5. Verify files were generated:
   ```bash
   for f in "invoice_list_${MONTH}.xlsx" "invoice_summary_${MONTH}.xlsx"; do
     if [ ! -f "$f" ]; then
       echo "ERROR: $f was not generated."
       exit 1
     fi
   done
   ```
6. Send the generated files as attachments:
   - **With `invoice_close`:** ZIP archive is auto-generated — send `invoice_${MONTH}.zip`
   - **With `invoice_export`:** Send individual xlsx files, or manually zip:
     ```bash
     cd <thread_dir>
     zip -r "invoice_${MONTH}.zip" "invoice_${MONTH}/" "invoice_list_${MONTH}.xlsx" "invoice_summary_${MONTH}.xlsx"
     ```
7. Send the zip file as an attachment in the reply

If the user asks for a specific month that doesn't exist, reply with available months:
```bash
ls -d invoice_*/
```

Or query the database for months with data:
```
invoice_list --month YYYY-MM  # or omit --month to list all
```
